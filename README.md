# AtliQ Hotels Data Analysis Project

This project is a comprehensive data analysis exercise performed on the hospitality domain. It involves cleaning, transforming, and analyzing multiple datasets related to hotel bookings, room categories, and properties to extract meaningful business insights.

*Note: This project was completed as part of a guided course by Codebasics. To respect data privacy policies, the source datasets are not included in this repository.*

## 🏨 Project Overview
The goal of this project was to analyze booking data for AtliQ Hotels to understand occupancy rates, revenue distribution, and platform performance.

### Key Learnings & Skills
- **Data Cleaning:** Handling invalid entries (e.g., negative guest counts) and removing outliers using standard deviation.
- **Data Transformation:** Merging multiple dataframes (bookings, properties, dates) and creating new metrics like `occupancy percentage`.
- **Insights Generation:** Grouping data by city, room category, and day type (weekday vs. weekend).
- **Visualization:** Using Matplotlib to visualize city performance and booking trends.

## 🛠️ Tools Used
- **Language:** Python
- **Library:** Pandas, Matplotlib, Jupyter Notebook

## 📊 Key Insights
- **City Performance:** Analyzed which cities (Delhi, Mumbai, Hyderabad, Bangalore) have the highest occupancy.
- **Room Categories:** Identified that Presidential Suites (RT4) generally maintain higher occupancy rates.
- **Booking Trends:** Weekend vs. Weekday analysis revealed significantly higher booking volume on weekends.

## 🚀 How to Run
Since the data is restricted, you can view the completed logic and visualizations by opening the `AtliQ_Hotels_Data_Analysis.ipynb` file directly in the GitHub browser.
